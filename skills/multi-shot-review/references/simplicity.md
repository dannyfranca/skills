# Simplicity Review

Read `task.md` and every explicitly referenced requirement source. Review whether the solution meets the requirement directly and elegantly.

- Identify scope creep, speculative generality, unnecessary layers, premature configurability, and machinery without a present requirement.
- Prefer the smallest design that fully handles required behavior and realistic failure modes.
- Do not mistake missing correctness, safety, or necessary domain modeling for simplicity.
- Treat registered related/future tasks as explicit exclusions from current scope.
